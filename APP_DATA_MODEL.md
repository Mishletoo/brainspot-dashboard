# План за структурата на данните — BrainSpot Dashboard

> Това е начален план. Не съдържа SQL код. Написан е на прост език, за да е разбираем за всеки.

---

## Техническа основа

Приложението ще бъде изградено с **Next.js** (Server-side first) и **Supabase** като база данни и бекенд услуга.
Това означава, че повечето логика ще се изпълнява на сървъра, а не в браузъра.

---

## 1. Основни модули на приложението

| Модул | Какво прави |
|---|---|
| **Потребители и роли** | Управление на акаунти и права за достъп |
| **Служители** | Информация за хората в компанията |
| **Заплати** | Следене на изплатени заплати и компенсации |
| **Проследяване на работното време** | Кой колко часове е работил и по какво |
| **Клиенти** | Фирми или хора, за които работим |
| **Проекти** | Задачи и ангажименти, свързани с клиенти |
| **Приходи** | Пари, постъпили от клиенти |
| **Разходи** | Пари, изразходвани от компанията |

---

## 2. Основни таблици в базата данни

### `users` — Потребители
Хората, които влизат в системата.

| Поле | Обяснение |
|---|---|
| `id` | Уникален номер |
| `email` | Имейл за вход |
| `full_name` | Пълно име |
| `role` | Роля (admin, manager, employee) |
| `created_at` | Дата на създаване |

---

### `employees` — Служители
Подробна информация за служителите на компанията.

| Поле | Обяснение |
|---|---|
| `id` | Уникален номер |
| `user_id` | Връзка към акаунта в `users` |
| `full_name` | Пълно име |
| `position` | Длъжност / позиция |
| `department` | Отдел |
| `start_date` | Дата на постъпване |
| `status` | Активен / неактивен |

---

### `salaries` — Заплати
История на изплатените заплати.

| Поле | Обяснение |
|---|---|
| `id` | Уникален номер |
| `employee_id` | Връзка към `employees` |
| `amount` | Сума |
| `currency` | Валута (BGN, EUR и т.н.) |
| `paid_at` | Дата на изплащане |
| `notes` | Бележки (бонус, аванс и т.н.) |

---

### `time_logs` — Работно време
Записи за отработените часове.

| Поле | Обяснение |
|---|---|
| `id` | Уникален номер |
| `employee_id` | Кой служител |
| `project_id` | По кой проект |
| `date` | Дата |
| `hours` | Брой часове |
| `description` | Какво е свършено |

---

### `clients` — Клиенти
Фирми или хора, с които работим.

| Поле | Обяснение |
|---|---|
| `id` | Уникален номер |
| `name` | Наименование |
| `contact_person` | Лице за контакт |
| `email` | Имейл |
| `phone` | Телефон |
| `status` | Активен / неактивен |

---

### `projects` — Проекти
Всеки проект е свързан с клиент.

| Поле | Обяснение |
|---|---|
| `id` | Уникален номер |
| `client_id` | Връзка към `clients` |
| `name` | Наименование на проекта |
| `description` | Кратко описание |
| `start_date` | Начална дата |
| `end_date` | Крайна дата |
| `status` | В процес / завършен / отменен |

---

### `revenues` — Приходи
Пари, получени от клиенти.

| Поле | Обяснение |
|---|---|
| `id` | Уникален номер |
| `client_id` | От кой клиент |
| `project_id` | По кой проект (незадължително) |
| `amount` | Сума |
| `currency` | Валута |
| `received_at` | Дата на получаване |
| `notes` | Бележки |

---

### `expenses` — Разходи
Пари, изразходвани от компанията.

| Поле | Обяснение |
|---|---|
| `id` | Уникален номер |
| `category` | Категория (наем, софтуер, заплати и т.н.) |
| `amount` | Сума |
| `currency` | Валута |
| `paid_at` | Дата на плащане |
| `project_id` | Свързан проект (незадължително) |
| `notes` | Бележки |

---

## 3. Връзки между таблиците

```
users ──────────── employees        (един потребител = един служител)
employees ─────── salaries          (един служител има много заплати)
employees ─────── time_logs         (един служител има много записи за работно време)
clients ────────── projects         (един клиент има много проекти)
projects ────────── time_logs       (един проект има много записи за работно време)
clients ────────── revenues         (един клиент носи много приходи)
projects ────────── revenues        (един проект може да генерира приходи)
projects ────────── expenses        (един проект може да има разходи)
```

---

## 4. Бележка за архитектурата

- Приложението ще е **Server-side first** с **Next.js App Router**
- Данните се пазят в **Supabase** (PostgreSQL база данни)
- Автентикацията се управлява от **Supabase Auth**
- Достъпът до данни се контролира чрез **роли** (`admin`, `manager`, `employee`)
- Всички заявки към базата данни се правят от **сървъра**, не от браузъра
